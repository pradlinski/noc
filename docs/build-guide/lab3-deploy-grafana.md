# Lab 3 – Deploy Grafana with Podman Quadlets

## Lab Summary

This lab deployed Grafana as the first production-style service in the NOC monitoring platform using Podman Quadlets. The deployment validated the container infrastructure established in previous labs, including Quadlets, the shared `noc` container network, persistent volumes, SELinux enforcement, and Git-based deployment.

The deployment process was also refined to use an idempotent deployment script that synchronizes Quadlet definitions from the Git repository to the host operating system.

---

# Objectives

At the completion of this lab, the following objectives were achieved:

* Deploy Grafana using a Podman Quadlet.
* Store persistent data in a Podman volume.
* Attach Grafana to the shared `noc` container network.
* Publish Grafana only on the management interface.
* Validate operation with SELinux enforcing.
* Establish a repeatable Git-based deployment workflow.
* Validate Quadlet-generated systemd services.

# Deployment Workflow

Grafana is deployed using the standard deployment workflow for the NOC platform.

```
Modify files in Git
        │
git status
        │
git commit
        │
sudo scripts/deploy-quadlets.sh
        │
systemctl daemon-reload
        │
Restart affected services
        │
Validate deployment
```

The Git repository remains the authoritative source of all Quadlet definitions.

System files under `/etc/containers/systemd` are never edited manually.

# Lessons Learned

## Quadlet Services

Quadlet services are generated by the Podman system generator.

They are **not enabled** using:

```
systemctl enable
```

Instead, they are generated automatically from the Quadlet definitions.

---

## Git is the Source of Truth

Configuration changes are always made in the repository.

The deployment script synchronizes the repository with the operating system.

Operational configuration should never be edited directly under:

```
/etc/containers/systemd
```

---

## Idempotent Deployment

The deployment process was enhanced to be idempotent.

The deployment script now:

* Detects changed Quadlets.
* Copies only modified files.
* Removes obsolete managed files.
* Restores SELinux labels.
* Reloads systemd.
* Restarts only affected services.
* Reports deployment status.

Repeated executions with no changes result in no modifications.

---

## Plugin Auto-Updates

Grafana attempted to update bundled plugins located inside the immutable container image.

These update attempts generated permission errors but did not affect Grafana operation.

Automatic updates for bundled plugins were disabled to eliminate unnecessary log messages and improve deployment consistency.

---

## SQLite

A single SQLite database lock message was observed during startup.

This was transient and automatically retried by Grafana.

No further action was required.
